# IGamingTest Backend

Проект демонстрирует реализацию backend-сервиса с использованием CQRS, Entity Framework Core, Hangfire и Polly для устойчивого взаимодействия с внешними API.

---

## Основные возможности

- Автоматическое получение данных через API  
  Механизм получения данных с внешнего API по расписанию с помощью Hangfire и собственной обёртки для удобного управления cron-задачами.

- Приём данных по HTTP  
  Обёртка над HTTP-сервером для приема входящих данных с использованием CQRS: разделение команд и запросов с соответствующими хендлерами.

- CQRS-архитектура  
  Бизнес-логика инкапсулирована в CQRS-хендлерах. Вызов фоновых Hangfire-задач организован через сервисный слой, что обеспечивает отделение логики от контроллеров и улучшает тестируемость.

- Политики повторных вызовов  
  Для надежности при работе с внешними API применяется Polly с настроенными стратегиями повторных попыток при ошибках сети и таймаутах.

- Универсальные репозитории  
  Репозитории реализованы с использованием Expression<Func<...>>, что позволяет создавать гибкие, переиспользуемые методы и минимизировать дублирование кода.

---

## Структура проекта

- **IGamingTest.Core** – бизнес-логика, модели, контракты  
- **IGamingTest.Infrastructure** – работа с базой данных и внешними сервисами  
- **IGamingTest.Web** – API, контроллеры и настройка приложения  
- **IGamingTest.Contracts** – контракты для взаимодействия между слоями

---

## Основные используемые технологии

- ASP.NET Core  
- Entity Framework Core  
- MediatR (CQRS)  
- Hangfire  
- Polly
